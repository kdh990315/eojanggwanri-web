# 어장관리 웹 소개 사이트 디자인

## 목적

이 웹사이트는 어장관리 앱 내부 화면의 디자인을 그대로 따르는 프로젝트가 아니다. 어장관리 앱을 외부 사용자에게 소개하고, 필요할 때 AI 어종 판별 페이지로 연결하는 독립적인 웹 소개 페이지다.

메인 페이지 `/`는 앱 소개가 중심이다. AI 어종 판별은 메인 페이지 안의 한 섹션에서 소개하고, 실제 사진 업로드와 분석은 `/ai-species`에서만 제공한다.

## 레퍼런스 방향

참고한 사이트:

- Wanted: 넓은 흰 여백, 얇은 내비게이션, 정보가 많은 화면에서도 정돈된 카드/리스트 구성
- 당근 서비스 소개: 큰 한국어 헤드라인, 서비스별 실제 이미지, 따뜻하지만 과하지 않은 설명
- Toss: 강한 첫 화면 타이포그래피, 단순한 내비게이션, 밝은 배경 위의 선명한 제품 이미지와 CTA

적용 원칙:

- 세 사이트의 구조와 감각을 참고하되 특정 브랜드의 UI를 복제하지 않는다.
- 앱 내부 디자인, 바다색 테마, 낚시 장식 그래픽을 기준으로 삼지 않는다.
- 소개 페이지는 마케팅 랜딩처럼 보이되 과장된 세일즈 페이지가 아니라 신뢰도 높은 제품 소개 페이지처럼 보여야 한다.

## 디자인 키워드

- 깨끗함
- 넓은 여백
- 큰 한국어 타이포그래피
- 얇은 구분선
- 실제 제품 가치 중심 이미지
- 낮은 장식성
- 명확한 CTA
- 모바일 우선 가독성

## 시각 톤

기본 톤:

- 배경은 흰색 또는 아주 옅은 회색을 기본으로 한다.
- 본문 텍스트는 거의 검정에 가까운 색을 사용한다.
- 보조 텍스트는 회색으로 낮은 위계를 만든다.
- CTA는 검정 또는 선명한 블루 한 가지를 사용한다.
- 낚시 앱이라고 해서 바다색, 파도, 물결, 어두운 해양 배경을 사용하지 않는다.

색상 기준:

```text
background: #ffffff
foreground: #191f28
surface: #f9fafb
surfaceStrong: #f2f4f6
mutedText: #6b7684
border: #e5e8eb
primary: #191f28
primaryForeground: #ffffff
accent: #3182f6
accentSoft: #eef6ff
danger: #d92d20
```

주의:

- 화면 전체가 한 가지 색감으로 보이면 안 된다.
- 포인트 컬러는 CTA, 선택 상태, 링크 hover 정도에만 사용한다.
- 그라데이션, 장식 orb, 과한 그림자, 파도 패턴은 사용하지 않는다.
- 카드 안에 또 다른 카드를 넣지 않는다.

## 타이포그래피

폰트:

- 시스템 산세리프를 사용한다.
- 한국어는 `Pretendard`, `Apple SD Gothic Neo`, `Noto Sans KR`, 시스템 기본 순으로 자연스럽게 fallback한다.
- 별도 웹폰트 로딩은 성능과 구현 복잡도를 고려해 MVP에서는 필수로 두지 않는다.

크기 원칙:

- Hero H1은 크고 단단하게 사용한다.
- 섹션 제목은 크지만 과장하지 않는다.
- 카드, 목록, FAQ 내부 제목은 작은 제목 체계를 사용한다.
- viewport width에 따라 글자 크기를 직접 스케일링하지 않는다.
- letter spacing은 기본값을 사용하고 음수 자간을 쓰지 않는다.

권장 계층:

```text
Hero H1: 48-72px desktop, 36-44px mobile
Section H2: 36-48px desktop, 28-34px mobile
Card title: 18-22px
Body: 16-18px
Caption: 13-15px
```

## 레이아웃

공통:

- 최대 본문 폭은 1120-1200px 사이를 기본으로 한다.
- 섹션은 충분한 상하 여백을 둔다.
- 섹션 간 구분은 배경색 변화 또는 얇은 구분선으로 처리한다.
- 복잡한 장식보다 텍스트, 실제 이미지, 명확한 구조를 우선한다.
- 반복 카드의 border radius는 8px 이하를 기본으로 한다.

Hero:

- 첫 화면에서 어장관리 앱의 정체성이 바로 보여야 한다.
- H1은 브랜드명 또는 앱의 핵심 가치가 드러나는 문장으로 둔다.
- 설명 문구는 조과 기록 앱의 가치를 구체적으로 말한다.
- CTA는 1차 버튼과 2차 버튼을 명확히 구분한다.
- 히어로 이미지는 장식용 일러스트보다 앱 사용 맥락, 제품 화면, 조과 기록 흐름을 보여주는 이미지가 우선이다.
- 첫 viewport에서 다음 섹션의 시작이 약간 보이도록 한다.

메인 페이지 섹션 순서:

1. Hero: 앱 소개와 핵심 가치
2. 조과 기록 가치: 왜 기록해야 하는지
3. 앱 기능 요약: 사용자가 실제로 얻는 기능
4. AI 어종 판별 소개: `/ai-species`로 이동하는 버튼 포함
5. 자주 찾는 어종 링크
6. FAQ 요약
7. 앱 다운로드 또는 준비 상태
8. Footer

## 내비게이션

구성:

- 좌측에는 `어장관리` 텍스트 로고를 둔다.
- 우측에는 주요 메뉴를 짧게 둔다.
- 메뉴 수는 4-5개를 넘기지 않는다.
- Header는 흰색 배경, 얇은 하단 border, 적당한 높이를 사용한다.
- 데스크톱에서는 CTA 버튼 하나를 노출할 수 있다.
- 모바일에서는 메뉴를 접되 CTA가 화면을 과하게 차지하지 않게 한다.

권장 메뉴:

- `조과 기록`
- `앱 기능`
- `AI 어종 판별`
- `FAQ`

## CTA

주요 CTA:

```text
AI 어종 판별하기
```

대상:

```text
/ai-species
```

보조 CTA:

```text
조과 기록 앱 알아보기
```

원칙:

- 메인 페이지 Hero의 1차 CTA는 앱 소개 흐름에 맞춰 정한다.
- AI 섹션의 `AI 어종 판별하기` 버튼은 반드시 `/ai-species`로 이동한다.
- 앱 다운로드 링크는 소개페이지에서만 노출한다.
- `/ai-species`에서는 분석 흐름을 방해하는 다운로드 CTA를 직접 노출하지 않는다.
- 버튼은 충분한 터치 영역과 명확한 focus 상태를 가진다.

## 메인 페이지 `/`

역할:

- 어장관리 앱 소개 메인 페이지
- 조과 기록과 낚시 노트의 가치를 이해시키는 페이지
- AI 어종 판별 페이지로 이동시키는 소개 섹션을 포함하는 페이지

Hero 방향:

- 앱 소개가 먼저다.
- "사진으로 물고기 이름 찾기"는 앱 가치 중 하나로 다룬다.
- H1은 앱의 기록 가치를 중심으로 작성한다.
- 히어로에 전체 AI 판별 UI를 넣지 않는다.

예시 톤:

```text
낚시의 순간을 기록으로 남기는 가장 쉬운 방법
```

AI 어종 판별 섹션:

- `/ai-species`에서 할 일을 짧게 설명한다.
- 사진 업로드 UI를 메인 페이지에 만들지 않는다.
- 결과 UI를 메인 페이지에 만들지 않는다.
- 흐름은 `사진 업로드 -> 영상 광고 1회 -> AI 후보 확인 -> 앱 가치 재소개`로 설명한다.
- `AI 어종 판별하기` 버튼을 제공한다.

조과 기록 섹션:

- 날짜, 위치, 물때, 날씨, 마릿수, 사진을 함께 남기는 가치를 보여준다.
- 실제 앱 화면 또는 앱 화면에 가까운 목업 이미지를 사용한다.
- 고급 통계는 MVP 이후 확장 가치로만 표현한다.

앱 기능 요약:

- 사진으로 조과 등록
- 직접 입력 조과 등록
- AI 어종 판별
- 물때/날씨/위치 기록
- 내 조과 목록과 상세
- 내가 잡은 어종 도감

주의:

- 커뮤니티, 랭킹, 결제, 고급 통계 화면을 주요 기능처럼 노출하지 않는다.
- 기능 설명은 짧게 쓰고, 이미지나 레이아웃으로 이해시키는 비중을 높인다.

## AI 어종 판별 페이지 `/ai-species`

역할:

- 실제 AI 어종 분석만 담당하는 페이지
- 검색 유입 사용자의 즉시 문제 해결
- 분석 결과 화면에서 어장관리 앱의 조과 기록 가치를 다시 소개

상태:

- 초기 상태
- 이미지 선택 상태
- 영상 광고 1회 노출 상태
- 광고 skip 또는 시간 종료 상태
- AI 판별 loading 상태
- 판별 결과 상태
- 실패 상태
- 재시도 상태

초기 화면:

- 제목: `AI 어종 판별`
- 설명: `물고기 사진을 올리면 AI가 어종 후보를 제안합니다. 결과는 참고용이며 직접 확인해 주세요.`
- CTA: `사진 업로드`

판별 결과:

- 후보 어종명
- 참고 신뢰도 또는 참고 지표
- 사용자 확인 안내
- `다른 사진으로 다시 판별`
- 결과 데이터를 보여주는 영역
- 어장관리 앱의 조과 기록 가치 재소개 영역

주의:

- AI 결과는 확정값이 아니라 후보로 표현한다.
- 100% 정확도, 자동 확정, 조과 보장 표현을 쓰지 않는다.
- 광고 클릭을 유도하지 않는다.
- `/ai-species`에 앱 다운로드 링크를 직접 노출하지 않는다.

## 이미지와 미디어

- 웹사이트와 앱 소개에는 시각 자산이 필요하다.
- 실제 앱 화면, 앱 화면 목업, 조과 기록 흐름을 보여주는 이미지가 우선이다.
- 어종 사진은 출처와 라이선스가 확인된 이미지로만 사용한다.
- 어두운 stock 이미지, 흐릿한 배경 사진, 낚시 감성만 있는 이미지는 피한다.
- 이미지는 실제 제품과 사용 맥락을 설명해야 한다.

## 접근성

- H1은 페이지당 하나만 사용한다.
- 버튼과 링크는 키보드 포커스가 명확해야 한다.
- 이미지 alt는 구체적으로 작성한다.
- 장식 이미지는 빈 alt를 사용한다.
- 색상만으로 상태를 구분하지 않는다.
- 오류 메시지는 다음 행동을 함께 안내한다.
- 모바일에서 버튼 텍스트가 줄바꿈되더라도 버튼 밖으로 넘치면 안 된다.

## 반응형

모바일:

- Hero는 한 화면 안에서 브랜드/핵심 가치/CTA가 보이게 한다.
- 2열 이상의 복잡한 그리드는 1열로 정리한다.
- 버튼은 엄지로 누르기 쉬운 크기를 유지한다.
- 고정 CTA는 실제 콘텐츠를 가리지 않을 때만 사용한다.

데스크톱:

- 넓은 여백과 강한 타이포그래피를 적극 사용한다.
- 이미지와 텍스트는 균형 있게 배치한다.
- 섹션마다 한 가지 메시지만 강조한다.

## 금지 표현

사용하지 않는다:

- `100% 정확한 어종 판별`
- `AI가 확정한 어종`
- `사진만 올리면 자동 저장`
- `광고를 클릭하면 판별`
- `광고 시청 없이 불가능`
- `광고 보상을 받으면 판별`
- 특정 어종 조과를 보장하는 표현

권장 표현:

- `AI가 어종 후보를 제안합니다.`
- `결과는 참고용이며 직접 확인해 주세요.`
- `사진으로 조과 기록 초안을 만들 수 있습니다.`
- `사진을 올리면 짧은 광고 후 AI 어종 판별이 시작됩니다.`
